# 微信使用指南 - 如何分享给侄女

## 📱 最简单的方法（推荐新手使用）

### 方法一：直接发送文件（5分钟搞定）

**步骤：**

1. **在电脑上操作**：
   - 找到 `gaokao-volunteer` 文件夹
   - 右键 → 发送到 → 压缩(zipped)文件夹
   - 得到 `gaokao-volunteer.zip` 文件

2. **通过微信发送**：
   - 打开电脑版微信或网页版微信
   - 找到侄女的聊天窗口
   - 将 `gaokao-volunteer.zip` 拖拽到聊天框
   - 点击发送

3. **侄女手机上操作**：
   - 在微信中接收文件
   - 点击下载并解压（长按文件→解压）
   - 打开解压后的文件夹
   - 点击 `index.html` 文件
   - 选择用浏览器打开
   - ✅ 开始使用！

**优点**：
- ✅ 不需要任何技术知识
- ✅ 不需要服务器和域名
- ✅ 离线也能使用
- ✅ 数据安全（不经过网络）

**缺点**：
- ⚠️ 每次更新需要重新发送文件
- ⚠️ 侄女需要知道如何解压和用浏览器打开HTML

## 🌐 进阶方法（适合长期使用）

### 方法二：部署到GitHub Pages（免费且稳定）

**前提条件**：
- 有GitHub账号（没有的话去 github.com 注册，免费）

**步骤（约10分钟）**：

1. **创建仓库**：
   - 登录 GitHub
   - 点击右上角 "+" → "New repository"
   - 仓库名填：`gaokao-volunteer`
   - 选择 Public（公开）
   - 勾选 "Add a README file"
   - 点击 "Create repository"

2. **上传文件**：
   - 进入刚创建的仓库页面
   - 点击 "uploading an existing file"
   - 把 `gaokao-volunteer` 文件夹里的所有文件拖进去
   - （不要上传文件夹本身，只上传里面的文件）
   - 点击 "Commit changes"

3. **开启Pages服务**：
   - 点击仓库页面的 Settings（设置）
   - 左侧菜单找到 Pages
   - Source 选择 "Deploy from a branch"
   - Branch 选择 "main"，文件夹选择 "/ (root)"
   - 点击 Save

4. **获取链接**：
   - 等待2-3分钟（会显示部署状态）
   - 页面顶部会出现访问地址：
     ```
     https://你的GitHub用户名.github.io/gaokao-volunteer/
     ```

5. **微信分享**：
   - 复制这个链接
   - 通过微信发给侄女
   - 侄女点击链接即可在微信内直接打开使用！✅

**优点**：
- ✅ 永久有效的链接
- ✅ 随时可以更新内容
- ✅ 侄女不需要下载任何东西
- ✅ 全球可访问（CDN加速）

**缺点**：
- ⚠️ 需要GitHub账号
- ⚠️ 首次配置需要10分钟左右

---

### 方法三：使用Vercel（更简单，也免费）

**步骤（约5分钟）**：

1. **准备代码**：
   - 把 `gaokao-volunteer` 文件夹压缩成zip

2. **部署到Vercel**：
   - 访问 [vercel.com](https://vercel.com)
   - 用GitHub账号登录（或注册）
   - 点击 "Add New" → "Project"
   - 选择 "Upload" 上传刚才的zip文件
   - 项目名称随便填，比如 `gaokao-helper`
   - 点击 Deploy

3. **获得链接**：
   - 30秒左右部署完成
   - 会显示一个 `xxx.vercel.app` 的域名链接
   - 复制这个链接发到微信即可✅

**优点**：
- ✅ 操作最简单（比GitHub还简单）
- ✅ 部署速度快
- ✅ 自动HTTPS（微信友好）

## 🔧 高级方法（有服务器的情况）

如果你有自己的云服务器（阿里云/腾讯云等）：

### 使用Nginx部署

```bash
# 1. 上传文件到服务器
scp -r gaokao-volunteer/* user@你的服务器IP:/var/www/html/gaokao/

# 2. SSH到服务器配置Nginx
ssh user@你的服务器IP

# 3. 创建Nginx配置文件
sudo nano /etc/nginx/sites-available/gaokao

# 4. 写入以下内容（替换成你的域名）
server {
    listen 80;
    server_name gaokao.你的域名.com;
    root /var/www/html/gaokao;
    index index.html;
    
    location / {
        try_files $uri $uri/ =404;
    }
}

# 5. 启用站点
sudo ln -s /etc/nginx/sites-available/gaokao /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

# 6. 申请HTTPS证书（微信要求）
sudo certbot --nginx -d gaokao.你的域名.com
```

完成后将 `https://gaokao.你的域名.com` 分享到微信即可。

## 💡 实用技巧

### 1. 如何让侄女更容易打开？

**如果用方法一（发文件）**：
- 告诉她："收到后点一下文件，然后找'用其他应用打开'，选浏览器"
- 可以录个10秒的小视频教她

**如果用方法二/三（链接）**：
- 直接发链接就行，微信会自动识别为网页
- 长按链接可以选择"在浏览器中打开"

### 2. 如何更新内容？

**方法一**：重新打包发送新版本

**方法二（GitHub/Vercel）**：
- 修改本地文件
- 推送到GitHub或重新上传Vercel
- 几分钟后自动更新，链接不变

### 3. 手机上直接测试？

启动本地服务器后：

1. 确保电脑和手机连接同一个WiFi
2. 在电脑命令行输入 `ipconfig`（Windows）找到IPv4地址（如192.168.1.100）
3. 手机浏览器打开：`http://192.168.1.100:8080`
4. 测试没问题后再正式部署

### 4. 微信分享卡片不显示预览？

这是因为缺少微信JS-SDK认证。对于个人使用影响不大，但如果有需要：

- 必须有已备案的域名
- 需要在[微信公众平台](https://mp.weixin.qq.com)注册公众号
- 获取AppID和AppSecret
- 配置JS-SDK（需要后端支持）

**建议**：个人使用可以先忽略这个问题，功能完全正常。

## ❓ 常见问题

### Q: 侄女打开后显示空白？
A: 
- 确认是用浏览器打开的（不是微信内置浏览器的问题）
- 如果是文件方式，确认所有文件都在同一文件夹
- 尝试换Chrome、Safari等主流浏览器

### Q: 提示"无法访问此网站"？
A:
- 如果是本地服务器：确认服务器正在运行，防火墙允许8080端口
- 如果是在线链接：检查网络连接，确认链接正确

### Q: 微信打不开链接？
A: 微信对非正规域名有限制。解决方案：
- 使用GitHub Pages或Vercel等正规平台
- 或直接发送文件让侄女用浏览器打开

### Q: 数据会丢失吗？
A: 
- 刷新页面后对话记录会清除（设计如此，保护隐私）
- 但可以随时重新开始新的分析
- 所有数据只在本地处理，不会上传

### Q: 可以同时多个人使用吗？
A: 
- 当前版本是单机版，每个人独立使用互不影响
- 但如果是同一个链接，数据不会共享（这是好事）
